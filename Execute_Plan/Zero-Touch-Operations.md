# 零人力运营极限挑战方案

> **诚实前提**：纯 0 人力不存在（平台规则 + 合同 + 学员服务不允许）。
> **现实目标**：把日常运营压到**每天 5 分钟**。
>
> 这份文档把前面的方案 A、方案 B、以及其他所有「降人力」配置串起来，给出一个**端到端的"半自动化工作流"**。

---

## 一、三档运营模式

| 档位 | 每日人力 | 关掉了什么审批 / 决策 | 风险 |
|------|---------|---------------------|------|
| 🟢 **谨慎档**（默认）| 15 分钟 | 每条内容、每条消息都人工批准 | 最低 |
| 🟡 **平衡档** | 5 分钟 | AI 内容合规通过后自动发、内部企微广播自动发 | 中（一次内容翻车由你赔）|
| 🔴 **激进档** | 0-2 分钟 | 全开自动 + 定时 + AI 发起 mission + 异常自动升级 | 高（要自信系统不会乱）|

本文档展示 🟡 **平衡档** 的完整配置，并给出 🔴 档的选项。

---

## 二、完整开关清单

### 2.1 自动审批白名单（方案 A）

在 `server/.env` 加 3 个开关：

```bash
# 🥇 内容自动发布：合规扫描通过 + 标题长度 5-50 字 + 平台合法 → 免审批
AUTO_APPROVE_CONTENT=true

# 🥈 内部企微广播自动发送：to=admins + 不含链接/手机号/微信号 → 免审批
AUTO_APPROVE_WECHAT_INTERNAL=true

# 🥉 线索安全状态迁移自动通过：仅 contacted / following 这种前向迁移 → 免审批
AUTO_APPROVE_LEAD_STATUS=true
```

**不开的（必须人工）**：
- `send_wechat_notice` 发给**外部学员**或**单一专员** → 仍需审批
- `update_lead_status` 迁到 `enrolled` / `lost` / `interested` → 仍需审批（影响漏斗）
- 退款 / 定金操作 → 仍需审批

### 2.2 AI 员工定时自启动（方案 B）

登录管理后台 → AI 员工 → 底部「定时自动化」卡片：

| 任务 | 默认时间 | 建议配置 |
|------|--------|---------|
| 每日内容冲刺 | 08:00 | ✅ 启用（早 8 点生成当天内容）|
| 线索跟进扫描 | 09:00 | ✅ 启用（早 9 点扫高意向）|
| 周度经营报表 | 周一 20:00 | ✅ 启用（周一晚推送上周总结）|

每 5 分钟 Worker 会检查一次，命中的立即入队。

### 2.3 其他降人力配置（已内置）

| 配置 | 默认 | 效果 |
|------|------|------|
| 私信雷达扫描 | 每 5 分钟 | 24/7 自动抓新私信，AI 意向分级，自动创建 lead |
| 合规采集 | 每 1 小时 | 自动采教育部 / 考试院政策喂给内容工厂 |
| RPA 账号健康检查 | 每 24 小时 | 登录态失效自动切 cooldown + 写风险提示 |
| 审计日志清理 | 每 24 小时保留 60 天 | 磁盘不会爆 |
| 作业队列清理 | 每 24 小时保留 7 天 | 历史作业自动归档 |
| 定金催付 | 留资后 1/3/7/14 天 | 自动走企微推送，已支付自动终止 |
| 内容效果回填 | 管理员每周一次 | 手工填 views/likes/leads 后完整漏斗可见 |
| 学员端主动留存 | 无（v3.2 尚未做）| 留到 v3.3 接入公众号模板消息 |

## 三、🟡 平衡档的完整日流程（每天 5 分钟）

### 早 8:00 · 全自动（你不用动）

1. Worker 定时器命中 `cronHour=8`
2. 创建 `daily_content_sprint` mission，goal 是默认的 3 条内容
3. Agent 拉院校素材库
4. Agent 调 Gemini 生成 3 条图文草稿
5. Agent 对每条调 `compliance_scan` → 全部 pass
6. Agent 调 `submit_content_for_review`
7. **方案 A 自动审批规则命中** → 免审批，直接入库
8. 内容状态立刻变 `pending` → 你手动审核后会自动排 RPA 任务
9. Agent 调 `finish_mission`，status = succeeded

### 早 9:00 · 全自动

1. Worker 命中 `lead_followup_sweep` 定时
2. Agent 查询高意向未跟进线索
3. 对每条调 `suggest_reply_script` 生成话术
4. Agent 调 `send_wechat_notice({ toUser: 'admins', content: '...' })` 推送给管理员
5. **方案 A 自动审批**（内容纯文本 + 发给 admins）→ 免审批直接发
6. 管理员的企业微信收到"今日高意向 TOP 10"推送

### 9:05 · 你花 3 分钟

1. 打开管理后台 → HomePanel
2. 看到顶部红色告警（如果有）：疑似绕单 / 审批积压
3. 进「内容工厂 → 待审核」
4. 看 3 条昨晚生成的内容，**只做一件事**：每条看一眼有没有逻辑问题
5. 全部点「通过」（或编辑后通过）→ 自动创建 RPA 发布任务
6. 关掉网页

### 12:00 · 系统自动

- 午间 RPA 任务触发（已登录的小红书号发布）
- 私信雷达扫一轮（12:00 刚好在 8-22 作息时段内）
- AI 意向分级 → 高意向自动创建线索

### 21:00 · 每周一次，你花 2 分钟

- 周一 20:00，Worker 命中 `weekly_report` 定时
- Agent 聚合 dashboard / deals / 疑似异常
- Gemini 生成自然语言周报
- 自动发到你企微
- 你扫一眼，决定下周是否需要调整（不需要就什么都不做）

### 半夜到早 8:00 · 静默

- 私信雷达继续扫（但作息时段外自动跳过，不会骚扰学员）
- 审计日志清理作业
- 健康检查作业

**全天你真的动手只有 9:05 的 3 分钟 + 周一 21:00 的 2 分钟 = 每周 5×3 + 2 = 17 分钟。**

## 四、🔴 激进档的额外配置（加完 = 0-2 分钟/天）

### 4.1 让 AI 员工自己发起 mission（未实现，v3.3 候选）

当前定时计划只能按固定时间触发。激进档想要的是：
> "凡是今日新线索 > 20 条，就自动触发 lead_followup_sweep"

**实现路径**：加一个 `metric-triggered mission` 机制，Worker 每小时检查指标是否跨阈值，命中就入队。

### 4.2 异常自动升级（未实现，v3.3 候选）

当前异常只是在 HomePanel 显示。激进档想要：
> "连续 2 天无新线索 → 自动给老板打电话（语音网关）"

**实现路径**：接阿里云语音 SDK + 异常检测作业。

### 4.3 把内容审核也自动化（危险）

当前 v3.2 的 `AUTO_APPROVE_CONTENT=true` 已经自动化**提交到审核队列**（content_items.status='pending'），但**审核动作本身**还是人工点的。

激进档再进一步：**审核通过自动化**：
```bash
# 不建议，但你要就加（需要改代码）
AUTO_APPROVE_CONTENT_AND_PUBLISH=true
```
这意味着 AI 生成 → 合规扫描 → 直接入 RPA 发布队列，零人工。

**⚠️ 风险**：一条不恰当内容会直接发到你的真实账号。出事了**系统没法兜底**，平台直接封号。

**建议**：这个开关**永远别开**。保留"人看一眼"是守住底线的最后一道关。

## 五、监控：零人力不等于无视

开了自动化 ≠ 不看系统。反而要**更密切地**看指标，因为错误不会直接 block 你：

### 每天必看（1 分钟）

```bash
# 一行命令看全部关键指标
curl -s http://服务器IP/api/metrics | grep -E "admissions_(jobs_total|ai_calls_last_day|rpa_accounts_logged_in|audit_writes)"
```

**红线**：
- `jobs_total{status="failed"}` 比昨天多 > 10 → 有东西坏了
- `ai_calls_last_day` 为 0 → Gemini 不响应或没人用
- `rpa_accounts_logged_in` 掉到 < 3 → 号被封或会话失效

### 每周一必看（5 分钟）

- AI 员工自动生成的周报（会推送到企微）
- 疑似绕单告警的数量
- 未结分成的金额

## 六、紧急刹车

如果某一刻发现 AI 乱跑（比如生成了离谱内容、在错时段发布），3 条紧急刹车：

### 刹车 1：关所有定时计划
登录管理后台 → AI 员工 → 定时自动化 → 所有开关都点关
系统立刻回到手动档。

### 刹车 2：关自动审批
```bash
cd /opt/admissions/Admissions_agents
sed -i 's/^AUTO_APPROVE_.*/AUTO_APPROVE_CONTENT=false/' .env
./docker/deploy.sh up
```
重启后所有 write_high 恢复「必须人工审批」。

### 刹车 3：停 Worker
```bash
docker compose stop worker
```
**Worker 停了 = agent 停了 + RPA 停了 + 定时停了**。主服务还在可以用（前端 + API）。
问题排查完再 `docker compose start worker`。

## 七、可检查的「零人力健康度」

给自己每月打一次分，9 项各 10 分：

- [ ] 开了 `AUTO_APPROVE_CONTENT`（+10）
- [ ] 开了 `AUTO_APPROVE_WECHAT_INTERNAL`（+10）
- [ ] 开了 `AUTO_APPROVE_LEAD_STATUS`（+10）
- [ ] 启用了定时 `daily_content_sprint`（+10）
- [ ] 启用了定时 `lead_followup_sweep`（+10）
- [ ] 启用了定时 `weekly_report`（+10）
- [ ] 私信雷达 `FETCH_DM_INTERVAL_MS` ≤ 10 分钟（+10）
- [ ] 过去 1 周 mission 成功率 > 80%（+10）
- [ ] 过去 1 周你真的只动过系统 ≤ 35 分钟（+20）

**90 分以上 = 真正的零人力运营。**
**70-89 分 = 平衡档跑顺了。**
**< 70 分 = 继续调。**

## 八、总结

| 你想要的 | 系统给你的 | 必须人做的 |
|---------|----------|----------|
| 全天 24 小时在跑 | ✅ Docker compose + Worker + RPA | 保证服务器不关机 |
| 每天自动生成内容 | ✅ 定时 daily_content_sprint + 自动审批白名单 | 审核内容 3 分钟 |
| 自动跟进线索 | ✅ 私信雷达 + lead_followup_sweep | — |
| 自动推送周报 | ✅ 定时 weekly_report | — |
| 自动处理异常 | ⚠️ 部分（出现告警但不会自动修） | 看告警做判断 |
| 零学员流失 | ❌ 教育服务的核心在人不在 AI | 自己带好课 |
| 零合同风险 | ❌ 签合同、开票、打款必须有人 | 财务 / 法务 |

**系统帮你处理的是「可标准化的重复劳动」。不可标准化的、合同相关的、学员服务的，必须保留人。**

一个合伙人 + 一套自动化系统 = 一个团队的产能。
这是 AI 数字员工真正的价值，不是「消灭人」，是「放大人」。
