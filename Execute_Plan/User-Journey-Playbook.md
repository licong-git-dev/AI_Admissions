## 1. 甲方老板 journey

**画像**：武汉聪可智核科技 · 李聪。合同第一方。关心 3 件事：
1. 这周赚了多少？（30% 分成计算结果）
2. 乙方有没有绕单？（合规审计）
3. 系统还活着吗？（RPA 账号 / Gemini API / 服务器）

### 典型一天

**早上 9:00** · 打开系统首页
- 期望：**一眼看到昨晚到今早发生了什么重要事**
- 现实：看到通用"数据看板"（今日新线索、已联系、意向明确、7 天趋势）
- **⚠️ 体验漏洞 #1**：老板要的是**「值不值得担心 + 值不值得开心」的摘要**，不是一堆数字图。

**早上 9:05** · 点「AI 员工」看昨晚 AI 干了啥
- 期望：有一份"AI 昨晚的战果"
- 现实：要一个个点开任务卡片看轨迹
- **⚠️ 体验漏洞 #2**：**缺 AI 数字员工的"日报"**，老板不关心每一步，关心"今天总共完成了什么"。

**早上 9:10** · 想看疑似绕单
- 期望：首屏直接告警
- 现实：去经营管理 → 审计日志 → 搜索...
- **⚠️ 体验漏洞 #3**：**异常信号没浮到首页**。

### 周五 18:00 · 周复盘
- 期望：下载本周对账单
- 现实：去平台总控台 → 手动算
- **⚠️ 体验漏洞 #4**：**缺"一键生成本周简报"**（可以直接发朋友圈/转发老板群）。

## 2. 乙方老板 journey

**画像**：云南德天惠军教育 · 邹晨曦。合同第二方。关心 3 件事：
1. 这个月招了多少学员？
2. 我的专员在不在干活？
3. 怎么让 AI 帮我多招一些？

### 第一次登录系统（接入第一天）
- 期望：系统告诉我"第一天该做什么"
- 现实：17 个菜单，不知道从哪开始
- **⚠️ 体验漏洞 #5**：**缺新手引导**。乙方接入首周就流失是最大风险。

### 日常使用
- 看专员业绩：要去**平台总控台或经营管理**，但**专员业绩排行在数据看板**
- **⚠️ 体验漏洞 #6**：**同类信息散落在 3 个地方**，信息架构不清晰。

- 想创建 AI 任务：AI 员工 → 新建任务。但乙方老板一般不懂技术，看到 JSON 参数就懵了
- **⚠️ 体验漏洞 #7**：**Mission 创建门槛过高**（让老板填 JSON 不合理）。

## 3. 招生专员 journey

**画像**：张三，兼运营和销售，每天要处理 30+ 条线索。

### 早上 9:00 · 上班第一件事
- 期望：系统告诉我"今天这 10 个人按顺序跟进"
- 现实：打开线索管理，看到全部租户的线索，手动过滤"我的"
- **⚠️ 体验漏洞 #8**：**没有"我的今日待办"**。

### 跟进中发现一个高意向学员
- 期望：点开 → 看到所有历史 + AI 推荐话术 + 一键发企微
- 现实：要先点「查看跟进」打开浮层，然后点「电话联系」才生成 AI 画像，再切到微信助手看话术
- **⚠️ 体验漏洞 #9**：**信息分散在 3 个 tab**，做一次跟进要切换 3 次。

### 下午 2:00 · 有 10 条待回复私信
- 期望：AI 帮我一键批量草拟
- 现实：要么一条一条点"AI 推荐话术"，要么去 AI 员工创建 mission（但 mission 是全局的，不是单条对话的）
- **⚠️ 体验漏洞 #10**：**批量 AI 处理的入口缺失**。

## 4. 学员 journey

**画像**：王小明，大专毕业 1 年，在小红书刷到笔记进测评。

### Step 1：点 H5 测评
- 答 5 题 → 填手机号 → 勾协议 → 提交
- 体验：✅ 流畅

### Step 2：拿到 AI 报告
- 看到"推荐专业 + 学校层次 + 时间规划 + 下一步建议"
- 下方：「锁定你的报名名额」500 元定金按钮
- **⚠️ 体验漏洞 #11**：**报告和定金之间缺"为什么要锁定"的说服逻辑**。直接让学员掏 500 块，转化率会很低。

### Step 3：学员直接关闭页面（90% 概率）
- 没有留微信、没有下载报告、没有后续触达路径
- **⚠️ 体验漏洞 #12**：**缺"温和的下一步"**。不是所有人都立刻掏 500 块，但应该有：
  - "加顾问微信领完整专业对比表 PDF"
  - "填微信公众号推送政策解读"
  - "7 天内可以免费预约 1 次顾问电话"

### Step 4（假如支付成功）：学员扫码登录学员端
- 打开 /portal，输入手机号 + 验证码
- 看到：状态 + 报读信息 + 缴费 + 定金 + 材料
- 但**学员期望**：下一步我要做什么？什么时候开课？有问题找谁？
- **⚠️ 体验漏洞 #13**：**学员端没有明确的"此刻的下一步"**。

### Step 5（一周后）：学员不会主动回来
- 当前系统没有主动 push 机制
- **⚠️ 体验漏洞 #14**：**学员端缺主动留存机制**（短信推送 / 企微群邀请 / 公众号绑定）。

## 6. 打磨优先级

### 本轮（v3.1）必做（5 项高价值低成本）

#### 🏆 打磨 1：**角色化 Home**（解决漏洞 #1 #2 #3 #8）
替换/增强现有 Dashboard，根据登录用户 role 显示不同首屏：

- **甲方老板（admin + tenant=platform）**：
  - 头部：【本周应收分成】【未结】【疑似绕单告警】【AI 员工今日产出摘要】
  - 中部：各乙方对比卡（线索 / 成交 / 分成 / 异常）
  - 底部：近 30 天趋势 + 系统健康

- **乙方老板（tenant_admin）**：
  - 头部：【本月成交】【待跟进】【今日新线索】【AI 员工任务状态】
  - 中部：专员业绩 + 渠道转化
  - 底部：7 天趋势

- **专员（specialist）**：
  - 头部：【今日我要跟进（Top 5）】【我的待催缴】【高意向新分给我的】
  - 中部：我的本周成交 / 跟进数
  - 底部：AI 推荐话术入口 + 常用动作

#### 🏆 打磨 2：**AI 任务快捷入口**（解决漏洞 #7 #10）
在现有工作流关键节点插入「让 AI 帮我」按钮：

- **线索列表顶部**：「让 AI 员工跟进 Top 20 高意向」（一键创建 lead_followup_sweep mission）
- **内容工厂顶部**：「让 AI 员工生成今日 3 条」（一键 daily_content_sprint）
- **经营管理顶部**：「让 AI 员工出本周周报」（一键 weekly_report）

免填 JSON，默认参数即可。

#### 🏆 打磨 3：**H5 测评「温和的下一步」**（解决漏洞 #11 #12）
报告页下方原来只有一个「锁定 500 元定金」按钮，升级为**3 档漏斗**：

- 🥇 **强转化**：「锁定 500 元定金名额」（不变）
- 🥈 **中转化**：「加顾问微信领完整 PDF 资料」（已有留资，引导企微二维码 / 电话）
- 🥉 **弱转化**：「关注公众号接收政策提醒」

用户不点 🥇 至少会点 🥈 或 🥉，留下持续触达的路径。

#### 🏆 打磨 4：**内容效果回流**（解决漏洞 #4）
发布任务成功后，content_items 增加「效果录入」入口：
- 手动/AI 填入浏览量 / 点赞 / 评论 / 从这条内容带来的线索数
- 前端内容卡片显示完整漏斗：发布 → 浏览 → 互动 → 线索 → 成交

#### 🏆 打磨 5：**乙方新手引导**（解决漏洞 #5）
乙方老板首次登录系统时，显示一个**入门 checklist**：
- [ ] 完善机构信息
- [ ] 录入至少 1 所院校 + 3 个专业
- [ ] 创建 1 个招生专员账号
- [ ] 登录 1 个 RPA 账号（任一平台）
- [ ] 创建第一个 AI 员工任务
- [ ] 查看 5 分钟入门视频（预留位置）

完成进度条 + 每项完成后立刻亮起✅。

### v3.2 候选（不在本轮做）
- 漏洞 #6：信息架构重组（工程量大）
- 漏洞 #9：线索详情页 Tab 内嵌 AI（改动面大）
- 漏洞 #13 #14：学员端主动触达（需对接公众号/短信，Phase 6）
- 漏洞 #2 的"AI 日报"自动生成：靠 weekly_report mission 扩展

## 7. 度量改进效果

上面 5 项打磨如果上线，预期可观察到：

| 指标 | 打磨前（估）| 打磨后目标 | 观测方式 |
|------|-----------|----------|---------|
| 专员每日平均使用时长 | 10-20 分钟 | 30-40 分钟 | 审计日志 user_id 活跃 |
| AI 员工任务创建频率 | 1-2 次/周 | 3-5 次/周 | agent_missions count |
| H5 测评 → 下一步动作 | 仅 5% 点击定金 | 至少 30% 进入某档漏斗 | lead_submissions 后续事件 |
| 乙方接入 7 天留存 | 未知 | 80%+ | users.last_login_at |
| 内容 → 线索归因可见率 | 约 40% | 90%+ | content_items.leads > 0 |

## 8. 写给产品经理的话

功能已经足够多了。每次想"再加一个功能"时，先问：

> **这个功能能让上面 14 个漏洞里的哪一个减少？**

不能的话，就是过度开发。
能的话，就做吧。
