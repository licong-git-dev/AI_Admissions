# 04 — 日常维护手册

## 每日必做（30分钟）

| 时间 | 任务 | 操作 |
|------|------|------|
| 早上9:00 | 审核AI生成的内容 | 系统 → 内容工厂 → 查看待审核 → 通过/修改/拒绝 |
| 早上9:15 | 查看新线索 | 系统 → 线索管理 → 筛选"今日新增" → 分配给招生专员 |
| 下午14:00 | 回复私信 | 各平台私信 + 微信消息，用系统AI生成话术辅助 |
| 晚上20:00 | 发布当天内容 | 审核通过的内容手动发布到各平台 |
| 晚上21:00 | 看数据看板 | 系统首页 → 今日线索/互动数据/待办事项 |

## 每周必做（周一，1小时）

| 任务 | 操作 |
|------|------|
| 复盘上周数据 | 数据看板 → 渠道转化率、专员业绩、内容效果 |
| 调整内容策略 | 哪类内容互动高？哪个平台转化好？下周重点打什么？ |
| 跟进滞留线索 | 筛选"已联系但超过7天未推进"的学员，安排回访 |
| 催缴提醒 | 缴费管理 → 筛选"逾期"和"即将到期"，发送提醒 |
| 补充素材库 | 有新的院校/专业/政策变化，及时更新 |

## 每月必做（月初，2小时）

| 任务 | 操作 |
|------|------|
| 月度数据报告 | 导出本月线索数、转化数、收入、各渠道ROI |
| 内容策略迭代 | 分析本月爆款内容特征，提炼成下月选题方向 |
| 竞品监控 | 看看同行在发什么内容、用什么话术 |
| 系统功能评估 | 哪些功能好用？哪些不够用？需要我帮你加什么功能？ |
| 院校信息更新 | 检查学费、政策、报名条件是否有变化 |

## 系统维护

### 服务器监控

```bash
# 查看服务器状态
top            # CPU和内存使用
df -h          # 磁盘空间
nginx -t       # Nginx配置检查

# 查看日志
tail -f /var/log/nginx/access.log   # 访问日志
tail -f /var/log/nginx/error.log    # 错误日志
```

### Gemini API 用量监控

- 免费额度：每分钟15次，每天1500次
- 如果超出：考虑升级到付费套餐（$0.075/百万字符输入）
- 监控地址：https://console.cloud.google.com/apis/dashboard

### 数据备份（后端上线后）

```bash
# 每日自动备份数据库（设置crontab）
0 3 * * * pg_dump admissions > /backup/admissions_$(date +%Y%m%d).sql

# 保留最近30天的备份
find /backup -name "admissions_*.sql" -mtime +30 -delete
```

### 常见问题排查

| 问题 | 可能原因 | 解决方法 |
|------|---------|---------|
| AI生成内容失败 | API Key过期或额度用完 | 检查.env中的GEMINI_API_KEY，检查API控制台用量 |
| 页面打不开 | Nginx未启动或配置错误 | `sudo systemctl restart nginx` |
| 数据丢失 | 当前还是Mock数据，无持久化 | 等后端部署完成后数据才会持久化 |
| 内容发布后无互动 | 发布时间不对/内容质量差 | 调整发布时段，优化选题 |

## 迭代计划

### 近期可以让我帮你做的改进

| 改进 | 工作量 | 价值 |
|------|--------|------|
| 后端API + 数据库 | 3-5天 | 数据持久化，多人使用 |
| 登录系统 + 权限控制 | 1-2天 | 区分管理员和招生专员 |
| 学员自助端 | 2-3天 | 学员自己查进度、传材料 |
| 企业微信集成 | 3-5天 | 自动客服+消息推送 |
| 内容批量导入/导出 | 1天 | Excel批量导入学员数据 |
| 微信公众号对接 | 2-3天 | 公众号自动回复+菜单 |

### 不建议现在做的

| 功能 | 原因 |
|------|------|
| RPA自动发布 | 先验证内容策略是否有效，再自动化 |
| 个人微信机器人 | 封号风险太高，先用人工+AI话术辅助 |
| 多平台同时铺 | 先在一个平台跑通，再复制 |

## 安全注意事项

1. **Gemini API Key 不要泄露** — 不要写在前端代码里暴露给用户，当前的Vite环境变量方式只适合自用，多人使用需要迁到后端
2. **学员隐私数据** — 手机号、身份证等信息需要加密存储（后端阶段实现）
3. **定期更换密码** — 管理员密码、服务器密码定期更换
4. **平台合规** — 不要在内容中使用"包过"、"100%通过"等违规用语，AI已经内置了这个规则
