# 02 — 数据初始化

## 为什么这步最重要

你的AI内容生成、话术推荐、意向分析，全都依赖基础数据。
数据没录，AI就是在编故事。

## 必须录入的数据（上线前完成）

### 1. 院校素材库

进入系统 → 系统设置 → 院校素材库，录入以下信息：

**每所院校需要录入：**

| 字段 | 示例 | 说明 |
|------|------|------|
| 院校名称 | 广东开放大学 | 全称 |
| 院校层次 | 开放大学 | 普通本科/211/985/开放大学/成人高校 |
| 招生类型 | 专升本、高起专 | 支持多选 |
| 院校简介 | 广东省属... | 100字以内，突出特色 |

**每个专业需要录入：**

| 字段 | 示例 | 说明 |
|------|------|------|
| 专业名称 | 会计学 | 全称 |
| 学费 | 28000 | 总学费，单位元 |
| 学制 | 2.5年 | 学习年限 |
| 通过率 | 95% | 如果没有精确数据，写行业平均值并标注 |
| 报名条件 | 大专及以上学历 | 简要说明 |
| 专业优势 | 就业面广，考公热门 | 2-3个卖点 |

**建议先录入你们最好卖的3-5所院校和对应专业。** 不用一次全录，后面慢慢补。

### 2. 机构基本信息

进入系统 → 系统设置 → 机构信息：

| 字段 | 必须填写 |
|------|---------|
| 机构名称 | 你的机构全称 |
| 联系电话 | 对外咨询电话 |
| 机构简介 | 一句话说清楚你是谁、做什么 |
| 核心优势 | 3-5个差异化卖点（通过率高、分期免息等） |

### 3. 招生专员账号

为每个招生专员创建账号（需后端部署后）：

| 字段 | 说明 |
|------|------|
| 姓名 | 真实姓名 |
| 手机号 | 用于登录和接收通知 |
| 角色 | 招生专员 |
| 负责方向 | 例如：专升本/MBA/某个院校 |

### 4. 已有学员数据迁移

如果你有Excel/微信聊天记录里的学员数据：

1. 整理成标准格式：姓名、手机、微信、意向专业、来源、当前状态
2. 通过学员管理页面逐个录入（或后续开发批量导入功能）
3. 先录入近3个月的活跃线索，老数据以后再补
