# MVP 每日操作 SOP · 14 天铁律

> 每天 3 个时段各 20-30 分钟。**总投入 < 1 小时/天**。
> 超时就是你在瞎忙，停下来检查 SOP 是不是被偷偷加戏。

---

## 每日时间表

```
08:00-08:30  内容生成 + 审核（30 分钟）
08:30        A 号发布
09:00-09:10  3 号同时检查：互动 / 评论 / 限流提示（10 分钟）
─────────────────────────────────
12:30        B 号发布
13:00-13:15  回复私信 + 评论（15 分钟）
─────────────────────────────────
20:30        C 号发布
21:00-21:30  数据收集 + 填表 + 微信培育（30 分钟）
```

---

## 每个动作的具体步骤

### 早上 08:00 · 内容生成（20-30 分钟）

**前 3 天**：从 [MVP-Content-Backlog.md](./MVP-Content-Backlog.md) 按顺序取今天要发的 3 条，直接喂给 Gemini

**Day 4-14**：基于前一天数据选题（见"每日选题决策"）

**步骤**：
1. 打开 https://aistudio.google.com/
2. 粘贴选题信息到 Saved Prompt
3. 3 次生成 3 条内容（3 个账号各 1 条）
4. 每条人工改 3 处：**标题加数字** / **开头加痛点** / **结尾加 CTA**
5. 把图文保存到桌面「今日内容」文件夹，命名格式：`2026-04-23_A_政策解读.md`

**品控红线**：
- ❌ 任何一处出现"包过 / 稳过 / 保录取" → 整条丢掉重生成
- ❌ 正文超 500 字 → 删到 400 字内（小红书长文完读率低）
- ❌ 3 条内容调性雷同 → 立刻停下换角度

---

### 上午 08:30 / 中午 12:30 / 晚上 20:30 · 发布

**步骤（每个账号都一样）**：
1. 打开小红书 App（**不要用浏览器发布**，容易被限流）
2. 新建笔记
3. 上传配图 1-3 张
4. 粘贴标题 + 正文 + tags
5. 位置：**不打位置**（跨地域引流更好）
6. 话题：**只选 1-2 个**（避免蹭热度嫌疑）
7. 发布

**发布之后立刻做**：
1. 截图保存笔记 URL 和发布时间
2. 填入 [MVP-Metrics-Template.csv](./MVP-Metrics-Template.csv)

---

### 上午 09:00 · 账号健康检查（10 分钟）

打开小红书后台分别看 3 个号：

| 检查项 | 正常 | 异常信号 |
|--------|------|---------|
| 主页能否正常访问 | ✅ | ❌ 点开显示"服务异常" |
| 昨天笔记浏览量走势 | 稳定增长 | 24h 后仍是 0 |
| 有没有系统通知 | 无通知 | 「违规提醒」「流量异常」 |
| 评论区有没有限流 | 评论可见 | 评论审核中但 1h 不通过 |

**发现异常账号处理**：
1. 当天**暂停发布**该账号
2. 去[MVP-XHS-14days.md](./MVP-XHS-14days.md) 红线章节对照
3. 第二天看能否恢复，不能恢复就**放弃这个账号**（不要硬救）

---

### 中午 13:00 / 晚上 21:00 · 私信 / 评论响应（15-30 分钟）

#### 评论区三级响应

**1. 真实问题（"学费多少"、"XX 专业怎么样"）**
- 在评论区**直接回答**（不要引到私信）
- 回答要具体：说出具体数字、具体条件
- 结尾追一句："有更具体问题在主页找我"

**2. 求资料（"求对比表"、"求政策原文"）**
- 评论回复："私信我'资料'两个字，发你"
- 私信里：给一段资料 + 问他想了解哪个专业

**3. 情感共鸣（"和我一模一样"、"加油"）**
- 真诚回一个表情 + 1 句话
- 不要拉扯，建立人设

#### 私信分流处理

| 等级 | 响应时间 | 动作 |
|------|---------|------|
| 高意向（问报名/价格/条件） | 30 分钟内 | 回答 + 引导加微信 |
| 中意向（泛咨询/对比） | 当天内 | 给资料 + 留微信口子 |
| 低意向（表情/一句话） | 24h 内 | 标准话术 |

#### 加微信话术

**私信版本**（高意向场景）：
```
同学你问的这个问题很具体，我在这里写几段你看不完。
我微信是 wx_xxxxxx（备注「小红书+专业名」），
加上我发你一份完整资料，还可以帮你具体分析适合哪个学校。
```

**评论版本**（不发微信号，引导私信）：
```
同学问的我有完整资料，私信你发～
```

### 晚上 21:00-21:30 · 数据收集 + 微信培育

#### 1. 填数据表（10 分钟）

打开 [MVP-Metrics-Template.csv](./MVP-Metrics-Template.csv)，填今天那一行的 4 个核心数字：

- [ ] 今日 3 条笔记合计浏览量
- [ ] 今日私信 + 评论数
- [ ] 今日加到微信的人数
- [ ] 今日有效线索数

#### 2. 微信培育（15 分钟）

**当天加到微信的新好友**：
1. 发"你好，我是小红书的 XX"（自我介绍）
2. 发"这是你要的资料" + 发 1 份 PDF（《专业对比表》或《政策白皮书》）
3. 问一个开放问题："你现在最想解决什么问题？"
4. **不在当天硬推销报名**

**昨天加的好友**：
1. 朋友圈点赞 1 条（建立存在感）
2. 不主动找话

**前天-大前天加的好友**：
1. 如果有新内容发朋友圈（朋友圈不是卖货，是展示专业）
2. 如果他问新问题，热情回答

**> 7 天未互动好友**：
1. 发 1 条"最近有 XX 政策变化，了解一下吗？"试探
2. 无回应就进入**长期培育**：朋友圈不主动打扰

#### 3. 记录今天的 3 个观察（5 分钟）

在 [MVP-Metrics-Template.csv](./MVP-Metrics-Template.csv) 最后一列「每日观察」填：

- 今天哪条笔记浏览最好？**为什么**好？
- 今天哪条笔记沉底？**为什么**沉？
- 收到的私信有什么共同问题？

这 3 个答案是**下周选题的金矿**。

## 每周一 09:00-09:15 · 周复盘 SOP

### 步骤（按顺序不跳）

1. 导出 [MVP-Metrics-Template.csv](./MVP-Metrics-Template.csv) 上周 7 天数据
2. 算 3 个比率：
   - 浏览 → 互动率 = 总私信数 / 总浏览量
   - 互动 → 加微率 = 总加微信数 / 总私信数
   - 加微 → 线索率 = 有效线索数 / 总加微信数
3. 对比下面的"健康值"判断：

| 指标 | 不健康 | 普通 | 健康 | 爆款 |
|------|-------|------|------|------|
| 互动率 | < 0.3% | 0.3-1% | 1-3% | > 3% |
| 加微率 | < 20% | 20-40% | 40-60% | > 60% |
| 线索率 | < 30% | 30-50% | 50-70% | > 70% |

4. 对最低的那个比率问"为什么"：

| 瓶颈 | 问题出在 | 下周改什么 |
|------|---------|-----------|
| 互动率低 | 内容本身没吸引力 | 换选题方向（从数据找爆款模式复刻） |
| 加微率低 | 私信话术没价值 | 重写"给资料"话术，资料要实在 |
| 线索率低 | 微信培育没到位 | 朋友圈内容不够专业、回复太慢 |

5. 把决定写一行到下周的 SOP 备忘。

## 异常处理预案

### 情况 A：Day 7 一条线索都没有

**不要骗自己"再等等"**。按这个顺序排查：
1. 3 条内容找一个懂小红书的朋友/同行看，问"你觉得这是真人写的吗"（如果他说"不像"，问题在内容质量）
2. 看 3 个号的**自然搜索**排名：搜"专升本报名"能否看到你的号（搜不到就是账号权重太低，可能需要先养号 1 周）
3. 检查有没有违规词导致限流（去小红书后台看「账号状态」）
4. 如果 Day 10 还是 0 条 → **提前结束**，走 No-Go 流程

### 情况 B：某个号爆了（单条浏览 > 10000）

**不要兴奋着加班**：
1. 当天还是按 SOP 做，不额外加码
2. 记录爆款笔记的 **3 个特征**（标题结构、开头 3 句、CTA 方式）
3. 下周在**那个爆款号**上严格复刻这 3 个特征
4. **千万不要**在另外两个号也模仿（会被判为矩阵搬运）

### 情况 C：某个号被平台警告

1. 该号**立即停发 3 天**
2. 删除警告对应的那条笔记
3. 3 天后发 2 条安全内容（政策解读类，最保守）
4. 观察 7 天无异常，才恢复常规节奏
5. **别心疼这个号**，有封号征兆的号后面不值得投入

## 每天问自己 3 个问题

1. 今天这 3 条内容，**我自己会不会看完**？不会 → 下一条改。
2. 今天加的 X 个人，**我记得他们各自的问题吗**？不记得 → 说明你没好好聊，明天别急着加新的。
3. **今天的 4 个数字诚实吗**？如果为了 KPI 好看虚报 → 你在骗你自己。

## 你不要做的事（再次提醒）

- 🚫 不要顺便注册抖音 / 快手 账号（聚焦是命根子）
- 🚫 不要"我再优化一下系统功能"（2 周内**不改代码**）
- 🚫 不要请人帮你发（你要的是数据诚实，别人帮发你看不见真相）
- 🚫 不要在笔记里承诺任何"通过率数字"（除非有真实数据支撑）
- 🚫 不要比较其他机构的账号（他们已经跑了几年，和你不在同一起跑线）
