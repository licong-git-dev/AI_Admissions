# Product Spec V3 · 路线图草案

> **当前版本**：v2.2（Phase 1-4 基本闭环 + Docker + CI + 监控 + 催付 + SMS）
> **本文档**：v3 是下一代产品形态，跨度 6-12 个月。不是承诺做，是**候选清单**。
>
> 所有 v3 候选方向都已经脱离"把合同跑通"的必须项，属于**扩张型**决策。
> 做不做、什么时候做，取决于 v2.x 跑出来的真实数据。

---

## 决策前提（不先答这 3 个问题，不做任何 v3）

1. **v2.x 系统稳定跑了 ≥ 6 个月了吗？** 没跑稳就想做新功能 = 在沙子上盖楼。
2. **签了 ≥ 3 家乙方了吗？** 只有 1-2 家的时候，每个 v3 特性都是按需定制，不是产品；签到 3 家才能看清楚需求的**共性**。
3. **有 2+ 人的研发和运营团队了吗？** v3 的每一个模块都是中等工程量（20-40 人天），一个人扛不动。

三个都 Yes 才动。

## v3 候选模块（按业务价值 × 技术可行性排序）

### 1. AI 电话外呼（VLA） ⬆ 优先级上调至 Q1 2027

**业务价值**：★★★★★（v2.3 竞品调研后上调）
**技术可行性**：★★★

#### 基于 2026-04 调研的新论据

- **成人教育转化窗口仅 3-7 天**（相比 K12 的数月），5 分钟首次触达是行业标配
- 市面成人教育大模型外呼选型普及：讯飞 / 阿里 / 腾讯都有成熟产品
- 某头部保险经纪单月 2000+ 线索从 KOS + AI 外呼结合打法获得
- 2026 是"AI 平权年"，语音合成成本从 2024 年的 0.8 元/分钟降到 0.3 元/分钟
- 招生顾问是稀缺资源：中小机构地推 2 小时只有 30 分钟真派单

**结论**：Q1 2027 提前落地（比原规划提前 3 个季度）。

#### 是什么
- 基于 Gemini Live API / 阿里云通义语音 / 讯飞星火（选一对接）
- 对高意向学员自动发起首次电话（不是真人）
- 脚本：自报家门 → 确认报名意向 → 引导加微信 → 若明确拒绝则自动挂断并更新线索状态
- 转化窗口短的场景价值最大：**留资后 5 分钟内第一次 AI 外呼触达**

#### 为什么有价值
- 合同 3.1 条要求招生专员"跟进转化"，但专员是稀缺资源
- AI 外呼是行业已验证的降本手段（K12、职业培训业界普遍使用）
- 对低意向学员只需要 30 秒即可过滤，节省 90% 的无效沟通时间

#### 风险 & 合规
- 🔴 **工信部《通讯短信息服务管理规定》** + 各地呼叫中心监管
- 🔴 必须明确标注「机器外呼」身份、必须提供拒接后永久屏蔽机制
- 🔴 白天时段限制（8:00-21:00），每号每日拨打上限
- 💰 语音合成 + 电信线路费用约 0.3-0.8 元/分钟

#### 实现骨架（若上）
- 新增 `call_tasks` 表（lead_id, scheduled_at, status, call_sid, recording_url, transcript, outcome）
- 新 Worker `call-worker/`：调用语音平台 SDK
- 录音 + 转写全流程入 `audit_logs`（电话业法律要求 3 年留存）
- 前端新增"电话中心"一级导航，展示呼叫记录 + 播放录音

#### Go / No-Go 门槛
- Go：已经有 200+ 高意向线索积压、真人专员无法及时跟进
- No-Go：线索量少或专员跟进延迟 < 2 小时

### 2. 长期学员运营（在读 → 毕业 → 复购）

**业务价值**：★★★★
**技术可行性**：★★★★★

#### 是什么
- 学员付费成交之后，系统持续陪伴 30 个月（本科学制）
- AI 自动推送：考试提醒、作业截止、毕业设计节点、就业辅导
- 毕业后引导报考研究生或同级别新产品（复购）

#### 为什么有价值
- 合同只涵盖"成交即结束"，但 **学员 LTV（终身价值）才是教培的真命题**
- 一个本科学员毕业后若复购硕士、在职 MBA，客单价从 3 万 → 8 万
- 系统已经有了学员完整画像 + 入学时间 + 报读专业，是天然优势

#### 实现骨架
- 新表 `student_lifecycle_events`（student_id, stage, scheduled_at, content_template, sent_at）
- 内置生命周期模板（预定义每学期节点）
- 新的 job handler `lifecycle.trigger`，每日巡检命中节点的学员并推送
- 前端经营管理新增"学员生命周期" Tab，展示每个学员的下一步节点
- 复购触发：毕业前 3 个月自动推送"硕士 / 职业技能证书"选项

#### Go / No-Go 门槛
- Go：已累计 50+ 已毕业学员（才有 LTV 数据）
- No-Go：学员池 < 30 人

### 3. 第三方 ERP 集成（金蝶 / 用友 / 企微会话存档）

**业务价值**：★★★
**技术可行性**：★★★★

#### 是什么
- 乙方的财务已经有 ERP 系统（金蝶云星辰 / 用友 T+）
- 系统把成交/分成数据自动同步到 ERP 做账
- 企业微信会话存档自动导入，合规留痕 + 沟通回溯

#### 为什么有价值
- 乙方不愿意双重录入（系统 + ERP）
- 合同 4.1 条"系统自动记录 → 作为对账依据"要想落地必须和乙方财务系统打通
- ERP 接入是"接入第 3/4 家乙方"的必答题

#### 实现骨架
- 新 `integrations` 表（tenant, provider, config_json, last_sync_at, last_error）
- 每个 provider 一个 adapter（`integrations/kingdee.ts` / `yonyou.ts` / `wechat-work-archive.ts`）
- 后台管理页：乙方管理员自助配置 API key / 选择同步字段
- 定时 job `integrations.sync` 每日把 deals 同步到 ERP

#### 风险
- ⚠️ ERP 接口文档稀少，每家 ERP 版本差异大
- ⚠️ 对接需要乙方 IT 配合（不是纯技术活）

#### Go / No-Go 门槛
- Go：已有 ≥ 2 家乙方明确要求集成
- No-Go：需求还是一对一，别浪费开发

### 4. BI 看板（数据分析 + 自助探索）

**业务价值**：★★★
**技术可行性**：★★★★

#### 是什么
- Grafana 只能展示预定义的指标（已在 v2.2 完成）
- BI 看板允许甲方管理员**自助拖拽**生成报表：
  - 不同平台的内容-线索转化漏斗
  - 不同专员的业绩趋势
  - 不同专业的成交分布
  - 不同时段的加微率

#### 为什么有价值
- 甲方/乙方老板每月都在问"为什么这个月业绩下滑"
- 每次都要研发人员 ad-hoc 查询浪费时间
- BI 是数据驱动决策的基础设施

#### 实现骨架 A（轻量）
- 用 Metabase（开源 BI）直连 SQLite（只读）
- 预制 20+ 问题模板（SQL 已写好）
- 一键嵌入到 PlatformConsole 的 iframe

#### 实现骨架 B（重度）
- 自己实现拖拽式 BI（参考 Superset 但定制）
- 数据仓库（ClickHouse / DuckDB）+ 去规范化 schema
- 工程量：3 人月以上

#### Go / No-Go 门槛
- Go：已有 ≥ 3 家乙方 + 甲方每周 ≥ 10 次 ad-hoc 查询
- No-Go：当前 Grafana 能解答 80% 问题

### 5. 知识库 + RAG（沉淀沟通经验）

**业务价值**：★★★★
**技术可行性**：★★★★

#### 是什么
- 系统已经存了大量 `followups` 和 `rpa_messages`
- 用 embedding + 向量检索，让新专员提问时 AI 自动召回「历史上谁遇到过类似问题，怎么解决的」
- 也可以把《招生简章》《政策文件》切片入库，AI 回答时自动引用

#### 为什么有价值
- 招生行业经验传承靠"老带新"，信息流失严重
- RAG 是把团队隐性知识显性化的最直接方式
- AI 意向分析 + 话术推荐的质量会因 RAG 显著提升

#### 实现骨架
- 新表 `knowledge_chunks`（id, source, text, embedding_json, metadata）
- 嵌入服务：Gemini text-embedding-004（已经在用 Gemini）
- 向量查询：SQLite + 暴力扫描（规模小时够用）或 vector extension
- 检索增强：`ai/recommend-scripts` 调用前先 RAG 召回 top-5 相似历史对话
- 前端：线索浮层新增"历史参考"tab，展示相似案例

#### 风险
- ⚠️ Embedding 费用随知识库规模线性增长（Gemini embedding 约 $0.025/1M tokens）
- ⚠️ RAG 效果严重依赖内容质量，乱七八糟的历史对话会污染结果

#### Go / No-Go 门槛
- Go：累计 followup 数 ≥ 5000 条、有明显的"新专员上手慢"的运营痛点
- No-Go：线索量还小，直接用 few-shot prompt 就够

## 不会做的方向（明确排除）

### ❌ 自研大模型 / 微调模型
直接调 Gemini / GPT-4o 足够。自研大模型的开发成本 ≥ 200 人月，ROI 极低。

### ❌ 扩展到 K12 / 留学 / 考研 等其他教培赛道
这是产品定位漂移。学历提升已经是 30 亿+ 市场，把这个赛道做透比跨赛道更赚钱。

### ❌ 无代码 / 低代码流程编排器
产品能力没收敛稳定前，任何"通用化"努力都是提前优化。

### ❌ 多语言 / 海外版
国内合规要求 + 流量结构不适用海外。

### ❌ App（iOS / Android 原生）
H5 + 微信小程序（未来可选）已经覆盖 99% 移动场景。原生 App 维护成本是 Web 的 3-5 倍。

## 决策框架（以后看到"想做什么"时用）

任何一个新想法进来，按下面 5 个问题排序：

1. **不做会死吗？** — 合同条款硬约束？监管压力？
2. **做了能 2 个月内看到收入增长吗？** — 付费意愿是最硬的验证
3. **是当前乙方的 Top 3 抱怨吗？** — 真实的客户要求，不是拍脑袋
4. **已有系统里能不能绕过？** — 能用已有能力拼起来的就别造新模块
5. **值不值得 20+ 人天？** — 小于 20 人天就别立项，直接让工程加塞；大于 20 人天需要正式过 v3 决策
