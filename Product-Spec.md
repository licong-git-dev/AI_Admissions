## 产品说明

这是一套面向学历提升机构（专升本、硕博连读）的智能招生系统，覆盖从内容生产、流量获取、线索跟进到学员管理的完整招生闭环。系统由四个协同工作的智能体模块组成：AI内容工厂负责自动生成并发布招生内容；线索抓取引擎自动监控各平台私信并引导加微信建联；微信AI助手辅助招生专员沟通转化；招生管理后台统一管理学员数据、报名流程、缴费和排课。目标用户为1-5人的小型学历提升机构，核心解决获客难、人手不足的问题。

**系统架构说明（v2.2 真实部署状态）**：
- **前端**（React 19 + Vite 6 + Tailwind 4）：管理后台 + 内容工厂 + 线索转化工作台 + 合规中心 + 经营管理 + 平台总控台 + H5 测评 + 学员自助端
- **主服务**（Node + Express + SQLite + better-sqlite3）：REST API + JWT + 25 个路由 + 审计中间件
- **Worker**（独立进程，可选 Playwright）：RPA 发布 + 私信雷达 + 采集器 + 账号健康自检 + 作业队列
- **部署**（Docker Compose）：server + worker + frontend（nginx） 三容器
- **监控**（可选）：Prometheus scrape + Grafana dashboard

## 功能需求

### 模块一：AI内容工厂

- 维护院校素材库：录入合作院校、专业方向、学费、学制、报名条件、通过率等基础信息
- AI自动生成招生图文内容：根据素材库信息，自动生成政策解读、专业推荐、学员案例、报名提醒、答疑科普五种类型的图文内容
- 平台风格自动适配：同一主题自动生成三个版本——小红书版（种草风、图文并茂）、抖音版（干货风、短平快）、快手版（接地气、口语化）
- 内容日历管理：AI根据招生季节、政策节点、热点事件自动规划发布排期
- 内容审核流程：AI生成内容后进入待审核队列，管理员审核通过后标记为"待发布"
- RPA自动发布：审核通过的内容由RPA按排期自动发布到对应平台账号（抖音、快手、小红书）
- 发布效果追踪：记录每条内容的浏览量、点赞、评论、私信数等互动数据
- AI内容优化建议：根据历史发布数据，AI分析哪类内容效果好，自动调整后续内容策略

### 模块二：线索抓取+建联

- 多平台私信监控：RPA定时扫描抖音、快手、小红书的私信和评论
- AI意向分级：AI分析私信/评论内容，自动判断用户意向等级（高/中/低）
  - 高意向：明确提到想报名、问价格、问条件
  - 中意向：咨询了解、犹豫中、对比中
  - 低意向：随便问问、不相关内容
- 自动回复：低意向用户由AI自动回复，引导关注账号
- 高意向用户建联流程：
  - 自动回复引导语，附带微信号或二维码
  - 同步推送通知给招生专员（包含用户画像、意向分析、对话记录）
  - 记录到招生管理后台的线索库
- 微信添加（双通道）：
  - 企业微信：通过官方API自动发送好友申请
  - 个人微信：通过RPA辅助添加，需人工确认

### 模块三：微信AI助手

- 企业微信自动应答：
  - 自动识别常见问题类型（学费、学制、报名条件、通过率、院校对比、政策解读）
  - 根据素材库信息生成准确回答
  - 超出知识范围的问题标记为"需人工介入"，推送给招生专员
- 个人微信话术辅助：
  - AI实时分析用户消息，生成推荐回复话术
  - 招生专员可一键复制或修改后发送
  - 不自动发送，所有消息由人工确认
- 电话沟通辅助：
  - 电话前：AI自动整理该学员的完整画像（来源渠道、咨询记录、意向专业、顾虑点、沟通历史）
  - 电话中：提供沟通要点清单和异议处理话术
  - 电话后：招生专员记录沟通结果，AI更新学员状态和跟进计划
- 自动跟进提醒：
  - AI根据学员状态和沟通历史，自动生成跟进计划
  - 定时提醒招生专员该联系哪个学员、该说什么

### 模块四：招生管理后台

- 学员数据管理：
  - 学员档案：姓名、手机、微信、学历、工作、意向专业、来源渠道
  - 状态流转：新线索 → 已联系 → 意向明确 → 已报名 → 已缴费 → 已录取 → 在读 → 已毕业
  - 标签系统：可自定义标签分类学员（如"价格敏感"、"急需学历"、"在职考公"）
- 报名流程管理：
  - 报名表单：在线填写报名信息
  - 材料收集：学历证明、身份证、照片等材料上传和审核
  - 报名进度追踪：每个环节的完成状态和时间节点
- 缴费管理：
  - 支持记录缴费金额、缴费方式（全款/分期）、缴费时间
  - 分期提醒：自动提醒未完成的分期款项
  - 收入统计：按院校、专业、招生专员维度统计收入
- 排课管理：
  - 课程安排：按院校/专业/批次管理课程时间表
  - 上课提醒：自动推送上课通知给学员
  - 考勤记录：记录学员出勤情况
- 数据看板（首页）：
  - 渠道转化率：各渠道（抖音/快手/小红书/其他）的线索数、转化数、转化率
  - 招生专员业绩：每人的跟进数、转化数、成交额排名
  - 整体运营数据：总线索数、总报名数、总收入、在读人数、毕业人数
  - 内容发布数据：AI生成数量、发布数量、各平台互动数据汇总

### 模块五：AI 获客引擎（Phase 2c 新增）

本模块将原 Phase 4「高风险自动化」中的合规部分前置到 Phase 2c，承担系统的主动获客职能。包含四个子能力：

#### 5.1 子模块 A — RPA 内容自动发布

- **账号矩阵**：每平台 1 个品牌主号 + 2 个人设号（学姐/顾问），共 9 个测试账号（抖音 3 + 快手 3 + 小红书 3）
- **发布平台优先级**：Sprint 1 先跑小红书 → Sprint 2 扩抖音 → Sprint 3 扩快手
- **发布节奏**：每平台每日 3-5 条，分 8:00-9:00、12:00-13:00、20:00-21:00 三个时段
- **反检测策略**：
  - Playwright 托管真实浏览器（非 headless）
  - 固定设备指纹（每个账号一台"虚拟设备"）
  - 动作间随机延迟 60-180 秒
  - 严格作息时段（仅 8:00-22:00 发布与互动）
  - 单账号单日操作次数上限（发布 ≤5、评论 ≤20、点赞 ≤50）
- **封号应急**：任一账号连续 3 天异常（验证码频发、流量断崖）自动降频并推送告警
- **审核前置**：所有发布内容必须通过「内容工厂」的审核通过状态后才能进入发布队列

#### 5.2 子模块 B — 平台私信 / 评论监控

- **监控范围**：仅监控甲方/乙方授权的自家账号收到的私信、评论、@消息，**不抓取平台其他用户的公开数据**
- **扫描频率**：每 30 分钟定时扫描一次
- **AI 意向分级**：复用现有 `analyze-intent` 接口（Gemini Flash）
- **自动处理策略**：
  - 低意向：AI 自动回复标准话术，不创建线索
  - 中意向：创建线索 + AI 回复引导关注
  - 高意向：创建线索 + 自动回复引导加微信 + 推送通知给专员
- **数据入库**：所有创建的线索必须记录来源平台账号 ID、原始消息内容、抓取时间，用于审计

#### 5.3 子模块 C — 公开信息采集

- **采集源范围**（白名单，不可扩展到其他源）：
  - 教育部官网及各目标省份教育考试院的招生政策公告
  - 合作院校官网的招生简章、专业目录、学费公示
  - 平台公开热点选题标题（仅标题和发布时间，不含用户身份信息）
- **合规约束**：
  - 严格遵守目标网站的 robots.txt 和访问频率限制
  - 单域名请求间隔 ≥ 10 秒
  - 采集数据仅用于内容工厂素材喂入，不用于外部传播
- **明确禁止**：严禁采集任何含有学生个人身份信息（姓名、手机、学校、专业、简历等）的数据，发现此类数据主动过滤丢弃

#### 5.4 子模块 D — 合规留资入口

- **Sprint 1 交付：H5 专业测评工具**
  - 用户答 5 道选择题（学历背景、目标方向、时间预算、预算区间、顾虑点）
  - AI 调用 `recommend-scripts` 生成个性化「专业匹配报告」
  - 获取报告前必须：填手机号 + 勾选「我已阅读并同意《个人信息授权书》《隐私政策》」
  - 报告生成后，线索自动入库，并记录授权时间戳、IP、勾选记录
- **Sprint 2 交付：白皮书领取**
  - 物料：《2026 专升本政策白皮书》《目标院校通过率数据手册》
  - 扫码关注公众号 + 填手机号 + 勾选授权 → 自动发送下载链接
- **Sprint 3 交付：公众号菜单裂变**
  - 公众号底部菜单挂工具入口
  - 老用户推荐新用户留资送优惠券/礼品
- **数据合规记录**：每条通过留资入口进入的线索，必须在 `consents` 表中保存授权勾选的完整快照（协议文本版本号、勾选时间、IP、UA）

### 模块六：合规与授权管理（Phase 2c 新增）

本模块为 AI 获客引擎与定金支付模块的合规基础设施，与 Phase 2a 并行建设。

- **隐私政策与用户协议**：
  - 版本化管理（每次变更生成新版本号），所有留资记录关联对应版本号
  - 学员自助端首次登录强制勾选
  - 甲方后台可编辑协议正文，变更后需学员重新确认
- **个人信息授权书**：
  - 明确列出收集哪些信息、用于什么用途、保存多久、学员权利（查询/撤回/删除）
  - 留资入口必须展示并强制勾选，不得默认勾选
- **数据主体权利响应**：
  - 学员自助端提供「导出我的数据」「删除我的账户」入口
  - 删除请求由甲方管理员 3 个工作日内处理，留痕
- **数据最小化**：
  - 留资入口仅收集必要字段（手机号 + 意向），其他信息由学员主动补充
  - AI 获客抓取的原始私信内容保留 90 天后自动脱敏
- **平台内容合规**：
  - AI 内容生成规则强制过滤违规词（"包过"、"100%通过"、"保录取"等）
  - 发布前二次校验，命中违规词直接打回不进入审核队列

## UI 布局

### 整体布局
左右两栏布局。左侧为固定导航栏，宽度 220px，深色背景；右侧为内容区，自适应宽度。顶部有通知栏，显示待处理事项和系统通知。PC端和移动端响应式设计，移动端导航栏折叠为汉堡菜单。

### 左侧导航栏
- 顶部：系统Logo + 机构名称
- 导航菜单（从上到下）：
  - 📊 数据看板（首页）
  - 📝 内容工厂（内容生成、内容日历、发布记录）
  - 🔍 线索管理（线索列表、意向分级、跟进记录）
  - 💬 微信助手（对话列表、话术库、沟通记录）
  - 👥 学员管理（学员档案、报名进度、材料审核）
  - 💰 缴费管理（缴费记录、分期管理、收入统计）
  - 📅 排课管理（课程表、考勤记录）
  - ⚙️ 系统设置（院校素材库、账号管理、RPA配置）
- 底部：当前用户头像 + 退出登录

### 右侧内容区 - 数据看板（首页）
- 顶部一行四个真实运营数据卡片：今日新线索 / 已联系线索 / 意向明确 / 待跟进学员
- 中部左侧：渠道转化漏斗图（各平台线索→联系→意向明确→报名的转化）

## 用户使用流程

### 管理员/老板日常流程
1. 登录系统，首页看数据看板，了解今日线索、本月业绩和待办风险
2. 直接点击首页待办卡片，进入待跟进或待催缴工作视图
3. 进入内容工厂，查看AI生成的待审核内容
4. 审核内容：通过 → 进入发布队列；不通过 → 标注修改意见，AI重新生成
5. 查看线索转化工作台，重点关注高意向线索的成交资产是否完整、报价是否已发送、催缴节点是否明确
6. 查看招生专员业绩，发现问题及时调整

### 招生专员日常流程
1. 登录系统，查看今日待跟进学员列表
2. 收到新线索推送通知（包含来源、意向分析）
3. 进入微信助手，查看AI推荐的沟通话术
4. 复制话术，在微信中与学员沟通
5. 对高意向线索生成报价单/方案单，填写推荐院校、推荐专业、学制、学费、缴费方式、分期建议、适合人群、风险提示
6. 一键复制方案发给顾问或在微信中发送给学员，后续可继续编辑并保存回显
7. 需要打电话前，查看AI整理的学员画像和沟通要点
8. 电话结束后，记录沟通结果，系统自动更新学员状态
9. 学员确认报名后，引导填写报名表、上传材料、完成缴费

### 学员自助流程
1. 收到招生专员发送的自助端链接
2. 手机号验证码登录
3. 查看报名进度、课程安排
4. 在线上传报名材料（身份证、学历证明等）
5. 查看缴费信息和缴费记录

### AI内容自动化流程（后台运行）
1. 系统每日根据内容日历自动调用AI生成当日内容
2. 生成内容进入审核队列，推送通知给管理员
3. 审核通过后，RPA在预设时间自动发布到对应平台
4. 发布后自动抓取互动数据，更新效果统计
5. AI根据效果数据优化后续内容策略

### 线索自动抓取流程（后台运行）
1. RPA每30分钟扫描一次各平台私信和评论
2. AI分析新消息，判断意向等级
3. 低意向：自动回复标准话术
4. 中/高意向：创建线索记录，推送通知给对应招生专员
5. 高意向：自动回复引导语，附带微信联系方式

## 应用场景

1. **日常获客**：系统每天自动生成3-5条招生内容，管理员早上花10分钟审核通过，RPA自动发布到三个平台。晚上查看数据看板，了解今日新增线索数量和来源。

2. **高意向学员跟进**：一位在职人员在小红书私信"专升本怎么报名"，系统自动判断为高意向，创建线索并推送给招生专员。招生专员打开微信助手，看到AI整理的学员画像和推荐话术，复制话术在微信上沟通。

3. **高意向线索报价转化**：学员明确询问价格、专业、毕业周期后，顾问进入线索详情生成报价单/方案单，填写推荐院校、专业、学制、学费、缴费方式、分期建议、适合人群、风险提示，一键复制后发给学员。若学员反馈变化，可在原方案基础上继续修改并保存。

3. **电话转化**：招生专员准备给一位犹豫中的学员打电话。打开学员详情页，AI已整理好：该学员关注的是XX大学会计专业，主要顾虑是学费和通过率，建议强调分期付款和95%通过率数据。电话后记录沟通结果，系统自动更新状态。

4. **报名缴费管理**：学员确认报名后，招生专员在系统中创建报名记录，学员收到自助端链接，自己上传身份证和学历证明照片。缴费后系统自动更新状态，分期学员到期前3天自动收到缴费提醒。

5. **招生季冲刺**：秋季招生季来临，管理员在内容日历中设置高频发布模式，AI自动增加发布频率。数据看板实时显示各渠道转化情况，管理员根据数据将预算和精力集中在转化率最高的平台。

6. **H5 测评留资闭环（Phase 2c 新增）**：用户在小红书刷到一篇由 AI 生成的政策解读笔记，点击文末短链进入 H5 测评，1 分钟答完 5 题后系统要求填手机号并勾选授权协议，提交后 AI 实时生成「专业匹配报告」。该学员手机号自动入库为线索，携带来源标签、授权版本号、IP、UA 时间戳等合规信息；招生专员立刻收到推送，从"你领取的测评报告里提到的会计专业"作为开场话术切入沟通。

7. **RPA 发布矩阵日常（Phase 2c 新增）**：甲方管理员在「AI 获客 → 发布矩阵」查看 9 个测试账号今日发布情况。系统显示小红书"学姐号"今日已发 4 条、到达上限；抖音"品牌号"检测到评论区验证码频发，自动降频为每日 2 条并推送风险告警。管理员点击告警查看详情，决定是否切换备用账号池。

8. **私信雷达自动建联（Phase 2c 新增）**：某学员在抖音对某条政策解读评论"学费能分期吗？"，30 分钟内被私信雷达扫到，AI 判为高意向，自动回复引导语附加微信，并创建线索推送给招生专员。专员打开微信助手时，该线索已带有完整的"来源平台评论原文 + AI 意向分析 + 推荐话术"。

### Phase 2c 范围补充（2026-04-22 确认）

#### Phase 2c 目标
将系统从「被动等线索」升级为「主动合规获客」，建立甲方可复制的获客引擎与合规基础设施，与 Phase 2a 变现闭环并行推进。

#### Phase 2c 核心闭环
1. 内容工厂审核通过的内容进入 RPA 发布队列，由独立 Worker 按排期发布到指定账号
2. 私信雷达定时扫描自家账号私信 / 评论 → AI 意向分级 → 自动处理或推送专员
3. 素材采集器按白名单自动抓取教育政策与院校信息 → 喂入内容工厂
4. H5 测评 / 白皮书 / 公众号裂变带来的留资，附带完整授权链路进入线索库
5. 所有获客链路的数据在合规中心可审计、可溯源

#### Phase 2c 技术架构变更

- **新增独立 RPA Worker 进程**：
  - 技术栈：Node.js + TypeScript + Playwright + node-cron
  - 通过 SQLite 共享数据库与主服务通信（新增 `rpa_tasks` 任务队列表）
  - 独立部署，主服务异常不影响 RPA，RPA 异常不拖垮主服务
  - 日志独立落盘 `server/logs/rpa-*.log`
- **新增授权 / 合规数据链路**：
  - `consents` 表：用户授权记录（phone, agreement_version_id, ip, ua, checked_at）
  - `agreements` 表：协议版本管理（type, version, content, created_at, is_active）
  - `data_requests` 表：学员发起的数据导出/删除请求
- **新增 RPA 相关数据模型**：
  - `rpa_accounts` 表：账号矩阵（platform, nickname, cookies_encrypted, status, daily_quota, device_fingerprint）
  - `rpa_tasks` 表：发布/回复/监控任务队列（account_id, type, payload, scheduled_at, status, result）
  - `rpa_messages` 表：抓取到的原始私信/评论（account_id, platform_msg_id, content, fetched_at, processed_status）
- **新增采集相关数据模型**：
  - `crawler_sources` 表：白名单采集源（domain, type, robots_rules, frequency_limit, is_enabled）
  - `crawler_items` 表：采集到的内容（source_id, title, url, content, crawled_at, fed_to_factory_at）
- **新增留资相关数据模型**：
  - `lead_forms` 表：留资入口配置（type, name, fields_json, agreement_version_id, is_active）
  - `lead_submissions` 表：留资提交记录（form_id, phone, answers_json, lead_id, consent_id, ip, ua）

#### Phase 2c 验收标准
- 可配置 9 个测试账号的账号矩阵，并按排期自动发布到小红书（Sprint 1）、抖音（Sprint 2）、快手（Sprint 3）
- 私信雷达可定时扫描自家账号的私信/评论并创建意向分级后的线索
- 素材采集器可按白名单自动抓取至少 3 类公开信息，并将结果推入内容工厂
- H5 专业测评工具上线，包含完整授权链路（协议展示、勾选、版本号留痕）
- 合规中心可查看所有授权记录、协议版本、数据请求，支持导出 CSV
- 违规词库生效，AI 生成内容命中违规词直接打回
- RPA Worker 为独立进程，主服务崩溃不影响发布队列，反之亦然
- 所有 AI 获客链路的关键操作写入 `audit_logs`，具备合规审计能力

#### Phase 2c 合规红线（硬性约束）
- ❌ 严禁开发任何爬取学生个人信息（姓名、手机号、学校、专业、简历等）的功能
- ❌ 严禁未经授权使用任何第三方平台的用户数据
- ❌ 严禁在 AI 生成内容中使用「包过」「100% 通过」「保录取」「稳过」等违规词
- ✅ 所有留资必须有完整的授权链路（协议版本号 + 勾选时间 + IP + UA）
- ✅ 所有 RPA 操作限于甲方/乙方授权的自家账号
- ✅ 所有采集限于白名单内的公开信息源

### Phase 1 范围补充（2026-04-11 确认）

#### Phase 1 目标
将当前“前端原型 + mock 数据 + 前端直连 AI”的演示系统，升级为可供招生团队试运行的最小可用系统。

#### Phase 1 核心闭环
1. 新线索进入系统并统一入库
2. 招生专员查看线索详情并获取 AI 意向分析
3. AI 生成推荐跟进话术，供人工复制使用
4. 招生专员记录跟进结果，并同步更新线索状态、下一步动作和下次跟进时间
5. 对高意向线索生成并保存报价单/方案单，支持复制给顾问并回显
6. 对意向明确线索执行最小报名推进：确认报读院校/专业、报名进度、负责人备注
7. 对已报名线索登记首付款/全款信息，并记录下次催缴时间
8. 首页看板显示真实的线索、报名与待催缴数据

#### Phase 1 后端要求
- 新增后端服务，负责：
  - 线索管理 API
  - 跟进记录 API
  - AI 代理调用 API
  - 首页看板聚合 API
- Gemini API Key 仅保存在后端，前端不再直接读取或注入密钥
- Phase 1 可采用轻量技术栈（Node.js + TypeScript + Express + SQLite）

#### Phase 1 数据模型（最小版本）
至少包含以下数据实体：
- 用户（管理员/主管/招生专员）
- 线索（来源平台、昵称、联系方式、意向等级、当前状态、负责人）
- 跟进记录（沟通方式、内容、下一步动作、下次跟进时间）
- 报价单/方案单记录（线索ID、推荐院校、推荐专业、学制、学费、缴费方式、分期建议、适合人群、风险提示、复制文案、最后更新时间）
- 报名推进记录（意向院校、意向专业、报名进度、备注）
- 缴费记录（应收金额、已收金额、缴费方式、首付款时间、下次催缴时间）
- AI 调用记录（分析场景、输入、输出、时间）

#### Phase 1 线索状态（最小版本）
- 新线索
- 已联系
- 跟进中
- 意向明确
- 已报名
- 已流失

#### Phase 1 前端改造要求
- 内容工厂、线索管理、微信助手、首页看板优先接入真实 API
- 原有 mock 数据仅用于开发兜底，不再作为正式数据来源
- 微信助手在 Phase 1 仅实现“AI 话术辅助 + 人工沟通记录”，不做真实微信自动发送
- 招生专员可在微信助手中复制 AI 推荐话术或手动输入沟通内容，并将该内容记录为线索跟进记录，形成沟通留痕
- 首页看板在 Phase 1 优先展示真实的线索运营指标，如今日新线索、已联系线索、意向明确线索、待跟进线索、负责人表现、线索来源分布、近 7 天线索/跟进趋势
- 收入、缴费、在读人数等经营指标在 Phase 2 对应模块完成真实数据化前，不作为 Phase 1 首页核心验收项

#### Phase 1 验收标准
满足以下条件即可视为完成：
- 可创建、查询、更新线索
- 可新增并查看跟进记录
- 可为高意向线索创建、编辑、查看报价单/方案单
- 方案单字段完整展示并可保存回显
- 支持一键复制完整方案文案给顾问
- 可为线索登记报名推进信息（院校、专业、报名阶段）
- 可登记首付款/全款与下次催缴时间
- AI 意向分析与话术推荐改为通过后端接口调用
- 首页核心指标改为真实线索运营数据统计
- 首页不再依赖 mock 的收入、在读学员等非 Phase 1 数据作为核心判断依据
- 前端构建产物中不再暴露 Gemini API Key
- 项目具备最小运行文档与基础测试能力

> ⚠️ **风险提示与合规红线（2026-04-22 更新）**：
>
> **平台封号风险（可控）**：
> - RPA 账号必须使用甲方/乙方专门的测试账号，与主营账号严格隔离
> - 单账号每日操作配额严格执行（发布 ≤5、评论 ≤20、点赞 ≤50）
> - 账号矩阵分散风险：每平台 1 主号 + 2 人设号，一号出事不影响全局
> - 真实浏览器指纹 + 随机延迟 + 严格作息时段，降低被识别为自动化的概率
> - 连续 3 天异常自动降频 + 告警，严重时切换备用账号
>
> **法律合规红线（不可越线）**：
> - 🔴 严禁任何形式的爬取学生个人信息（姓名、手机号、学校、专业、简历）— 涉嫌《刑法》253 条之一「侵犯公民个人信息罪」
> - 🔴 严禁使用非授权手段获取第三方平台用户数据
> - 🔴 严禁在内容中使用「包过」「100% 通过」「保录取」等违规承诺用语 — 涉嫌虚假宣传
> - ✅ 获客链路必须走合规留资入口（H5 测评 / 白皮书 / 公众号），每条线索有完整授权链路
> - ✅ 私信监控仅限自家账号，不抓取其他用户公开数据
> - ✅ 个人微信部分坚持「AI 生成话术 + 人工发送」，不做自动发送
>
> **甲方/乙方责任边界**：
> - RPA 测试账号由甲方统一注册与管理，封号损失由甲方承担
> - 合规授权与协议正文由甲方法务审核，乙方使用前签署《合规使用承诺书》
> - 学员个人信息的安全存储、删除响应、导出请求，甲方作为数据处理者承担主要责任
