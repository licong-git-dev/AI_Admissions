# 招生智能体集群 - Product Spec

## 产品说明

这是一套面向学历提升机构（专升本、硕博连读）的智能招生系统，覆盖从内容生产、流量获取、线索跟进到学员管理的完整招生闭环。系统由四个协同工作的智能体模块组成：AI内容工厂负责自动生成并发布招生内容；线索抓取引擎自动监控各平台私信并引导加微信建联；微信AI助手辅助招生专员沟通转化；招生管理后台统一管理学员数据、报名流程、缴费和排课。目标用户为1-5人的小型学历提升机构，核心解决获客难、人手不足的问题。

**系统架构说明**：本系统由两部分组成：
- **Web管理平台**（可用 Google AI Studio Builder 构建）：招生管理后台 + AI内容生成界面 + 数据看板 + 学员自助端
- **自动化引擎**（需独立开发）：RPA社交平台发布 + 多平台私信监控 + 微信自动化

## 功能需求

### 模块一：AI内容工厂

- 维护院校素材库：录入合作院校、专业方向、学费、学制、报名条件、通过率等基础信息
- AI自动生成招生图文内容：根据素材库信息，自动生成政策解读、专业推荐、学员案例、报名提醒、答疑科普五种类型的图文内容
- 平台风格自动适配：同一主题自动生成三个版本——小红书版（种草风、图文并茂）、抖音版（干货风、短平快）、快手版（接地气、口语化）
- 内容日历管理：AI根据招生季节、政策节点、热点事件自动规划发布排期
- 内容审核流程：AI生成内容后进入待审核队列，管理员审核通过后标记为"待发布"
- RPA自动发布：审核通过的内容由RPA按排期自动发布到对应平台账号（抖音、快手、小红书）
- 发布效果追踪：记录每条内容的浏览量、点赞、评论、私信数等互动数据
- AI内容优化建议：根据历史发布数据，AI分析哪类内容效果好，自动调整后续内容策略

### 模块二：线索抓取+建联

- 多平台私信监控：RPA定时扫描抖音、快手、小红书的私信和评论
- AI意向分级：AI分析私信/评论内容，自动判断用户意向等级（高/中/低）
  - 高意向：明确提到想报名、问价格、问条件
  - 中意向：咨询了解、犹豫中、对比中
  - 低意向：随便问问、不相关内容
- 自动回复：低意向用户由AI自动回复，引导关注账号
- 高意向用户建联流程：
  - 自动回复引导语，附带微信号或二维码
  - 同步推送通知给招生专员（包含用户画像、意向分析、对话记录）
  - 记录到招生管理后台的线索库
- 微信添加（双通道）：
  - 企业微信：通过官方API自动发送好友申请
  - 个人微信：通过RPA辅助添加，需人工确认

### 模块三：微信AI助手

- 企业微信自动应答：
  - 自动识别常见问题类型（学费、学制、报名条件、通过率、院校对比、政策解读）
  - 根据素材库信息生成准确回答
  - 超出知识范围的问题标记为"需人工介入"，推送给招生专员
- 个人微信话术辅助：
  - AI实时分析用户消息，生成推荐回复话术
  - 招生专员可一键复制或修改后发送
  - 不自动发送，所有消息由人工确认
- 电话沟通辅助：
  - 电话前：AI自动整理该学员的完整画像（来源渠道、咨询记录、意向专业、顾虑点、沟通历史）
  - 电话中：提供沟通要点清单和异议处理话术
  - 电话后：招生专员记录沟通结果，AI更新学员状态和跟进计划
- 自动跟进提醒：
  - AI根据学员状态和沟通历史，自动生成跟进计划
  - 定时提醒招生专员该联系哪个学员、该说什么

### 模块四：招生管理后台

- 学员数据管理：
  - 学员档案：姓名、手机、微信、学历、工作、意向专业、来源渠道
  - 状态流转：新线索 → 已联系 → 意向明确 → 已报名 → 已缴费 → 已录取 → 在读 → 已毕业
  - 标签系统：可自定义标签分类学员（如"价格敏感"、"急需学历"、"在职考公"）
- 报名流程管理：
  - 报名表单：在线填写报名信息
  - 材料收集：学历证明、身份证、照片等材料上传和审核
  - 报名进度追踪：每个环节的完成状态和时间节点
- 缴费管理：
  - 支持记录缴费金额、缴费方式（全款/分期）、缴费时间
  - 分期提醒：自动提醒未完成的分期款项
  - 收入统计：按院校、专业、招生专员维度统计收入
- 排课管理：
  - 课程安排：按院校/专业/批次管理课程时间表
  - 上课提醒：自动推送上课通知给学员
  - 考勤记录：记录学员出勤情况
- 数据看板（首页）：
  - 渠道转化率：各渠道（抖音/快手/小红书/其他）的线索数、转化数、转化率
  - 招生专员业绩：每人的跟进数、转化数、成交额排名
  - 整体运营数据：总线索数、总报名数、总收入、在读人数、毕业人数
  - 内容发布数据：AI生成数量、发布数量、各平台互动数据汇总

## UI 布局

### 整体布局
左右两栏布局。左侧为固定导航栏，宽度 220px，深色背景；右侧为内容区，自适应宽度。顶部有通知栏，显示待处理事项和系统通知。PC端和移动端响应式设计，移动端导航栏折叠为汉堡菜单。

### 左侧导航栏
- 顶部：系统Logo + 机构名称
- 导航菜单（从上到下）：
  - 📊 数据看板（首页）
  - 📝 内容工厂（内容生成、内容日历、发布记录）
  - 🔍 线索管理（线索列表、意向分级、跟进记录）
  - 💬 微信助手（对话列表、话术库、沟通记录）
  - 👥 学员管理（学员档案、报名进度、材料审核）
  - 💰 缴费管理（缴费记录、分期管理、收入统计）
  - 📅 排课管理（课程表、考勤记录）
  - ⚙️ 系统设置（院校素材库、账号管理、RPA配置）
- 底部：当前用户头像 + 退出登录

### 右侧内容区 - 数据看板（首页）
- 顶部一行四个真实运营数据卡片：今日新线索 / 已联系线索 / 意向明确 / 待跟进学员
- 中部左侧：渠道转化漏斗图（各平台线索→联系→意向明确→报名的转化）
- 中部右侧：招生专员业绩排行榜（表格，列：姓名、线索数、跟进数、意向明确、已报名）
- 底部左侧：近7天线索与跟进趋势图（线索数 vs 跟进数）
- 底部右侧：真实运营待办与来源分布（待跟进学员、待催缴提醒、已发布内容、内容带来线索、线索来源分布）
- 待办事项卡片支持点击直达对应工作视图：
  - 点击“待跟进学员” → 打开线索转化工作台，并自动筛到待跟进线索
  - 点击“待催缴提醒” → 打开缴费管理，并自动筛到待催缴/分期学员

### 右侧内容区 - 内容工厂
- 顶部Tab：内容生成 / 内容日历 / 发布记录
- 内容生成页面：
  - 左侧40%：AI生成控制面板
    - 内容类型下拉选择：政策解读 / 专业推荐 / 学员案例 / 报名提醒 / 答疑科普
    - 目标平台多选：抖音 / 快手 / 小红书
    - 关联院校/专业下拉选择
    - 补充要求文本框（可选）
    - 「AI生成内容」按钮，居中，主色调
  - 右侧60%：生成结果预览
    - 按平台分Tab展示（抖音版 / 快手版 / 小红书版）
    - 每个Tab内显示标题、正文、建议配图描述
    - 底部按钮组：「编辑修改」「重新生成」「提交审核」

### 右侧内容区 - 线索管理（线索转化工作台）
- 顶部：搜索 + 筛选 + 排序栏
  - 搜索：支持按用户昵称、联系方式、最新消息关键词、负责人搜索
  - 筛选：来源平台、意向等级、跟进状态、待跟进、待催缴
  - 排序：默认按“待跟进优先”排序，可切换到“最新线索”“高意向优先”
- 主体：线索列表表格
  - 列：来源平台、用户昵称、意向等级（色标）、最新消息摘要、跟进状态、负责人、下次动作、下次跟进时间、操作
  - 待跟进线索在列表顶部高亮展示，方便招生专员先处理最紧急工作
  - 点击行展开详情：完整对话记录、AI意向分析、成交资产、跟进历史
- 右侧浮层（点击“查看/跟进”触发）：
  - 学员画像卡片
  - AI推荐话术
  - 成交资产区：诊断结论、推荐院校/专业、报价摘要、下次动作、催缴节点
  - 报价单/方案单卡：
    - 字段包含：推荐院校、推荐专业、学制、学费、缴费方式、分期建议、适合人群、风险提示
    - 支持保存方案、复制给顾问、标记已发送
    - 再次打开线索详情时回显最近一次保存内容
  - 报名推进与缴费登记区
  - 跟进记录输入框
  - 状态变更下拉

### 右侧内容区 - 微信助手
- 左侧30%：对话列表（按最新消息排序，显示未读数）
- 右侧70%：
  - 上方60%：对话详情（消息气泡样式）
  - 下方40%：AI辅助面板
    - AI推荐回复（1-3条候选话术）
    - 每条话术有「复制」「编辑后复制」按钮
    - 学员画像摘要（意向专业、顾虑点、沟通阶段）

### 右侧内容区 - 学员管理
- 顶部：搜索框 + 筛选（状态、院校、专业、标签）
- 主体：学员卡片列表或表格视图（可切换）
  - 卡片视图：显示姓名、状态标签、意向专业、最近沟通
  - 表格视图：列式展示全部字段
- 点击学员进入详情页：
  - 左侧：基本信息表单（可编辑）
  - 右侧上：状态流转时间线
  - 右侧下：沟通记录 + 缴费记录 + 材料清单

### 学员自助端（独立页面，移动端优先）
- 登录页：手机号 + 验证码登录
- 首页：显示当前状态（已报名/已缴费/在读等）
- 报名进度：各环节完成状态的进度条
- 缴费信息：已缴金额、待缴金额、缴费记录
- 课程安排：近期课程时间表
- 材料上传：拍照上传所需材料

## 用户使用流程

### 管理员/老板日常流程
1. 登录系统，首页看数据看板，了解今日线索、本月业绩和待办风险
2. 直接点击首页待办卡片，进入待跟进或待催缴工作视图
3. 进入内容工厂，查看AI生成的待审核内容
4. 审核内容：通过 → 进入发布队列；不通过 → 标注修改意见，AI重新生成
5. 查看线索转化工作台，重点关注高意向线索的成交资产是否完整、报价是否已发送、催缴节点是否明确
6. 查看招生专员业绩，发现问题及时调整

### 招生专员日常流程
1. 登录系统，查看今日待跟进学员列表
2. 收到新线索推送通知（包含来源、意向分析）
3. 进入微信助手，查看AI推荐的沟通话术
4. 复制话术，在微信中与学员沟通
5. 对高意向线索生成报价单/方案单，填写推荐院校、推荐专业、学制、学费、缴费方式、分期建议、适合人群、风险提示
6. 一键复制方案发给顾问或在微信中发送给学员，后续可继续编辑并保存回显
7. 需要打电话前，查看AI整理的学员画像和沟通要点
8. 电话结束后，记录沟通结果，系统自动更新学员状态
9. 学员确认报名后，引导填写报名表、上传材料、完成缴费

### 学员自助流程
1. 收到招生专员发送的自助端链接
2. 手机号验证码登录
3. 查看报名进度、课程安排
4. 在线上传报名材料（身份证、学历证明等）
5. 查看缴费信息和缴费记录

### AI内容自动化流程（后台运行）
1. 系统每日根据内容日历自动调用AI生成当日内容
2. 生成内容进入审核队列，推送通知给管理员
3. 审核通过后，RPA在预设时间自动发布到对应平台
4. 发布后自动抓取互动数据，更新效果统计
5. AI根据效果数据优化后续内容策略

### 线索自动抓取流程（后台运行）
1. RPA每30分钟扫描一次各平台私信和评论
2. AI分析新消息，判断意向等级
3. 低意向：自动回复标准话术
4. 中/高意向：创建线索记录，推送通知给对应招生专员
5. 高意向：自动回复引导语，附带微信联系方式

## 应用场景

1. **日常获客**：系统每天自动生成3-5条招生内容，管理员早上花10分钟审核通过，RPA自动发布到三个平台。晚上查看数据看板，了解今日新增线索数量和来源。

2. **高意向学员跟进**：一位在职人员在小红书私信"专升本怎么报名"，系统自动判断为高意向，创建线索并推送给招生专员。招生专员打开微信助手，看到AI整理的学员画像和推荐话术，复制话术在微信上沟通。

3. **高意向线索报价转化**：学员明确询问价格、专业、毕业周期后，顾问进入线索详情生成报价单/方案单，填写推荐院校、专业、学制、学费、缴费方式、分期建议、适合人群、风险提示，一键复制后发给学员。若学员反馈变化，可在原方案基础上继续修改并保存。

3. **电话转化**：招生专员准备给一位犹豫中的学员打电话。打开学员详情页，AI已整理好：该学员关注的是XX大学会计专业，主要顾虑是学费和通过率，建议强调分期付款和95%通过率数据。电话后记录沟通结果，系统自动更新状态。

4. **报名缴费管理**：学员确认报名后，招生专员在系统中创建报名记录，学员收到自助端链接，自己上传身份证和学历证明照片。缴费后系统自动更新状态，分期学员到期前3天自动收到缴费提醒。

5. **招生季冲刺**：秋季招生季来临，管理员在内容日历中设置高频发布模式，AI自动增加发布频率。数据看板实时显示各渠道转化情况，管理员根据数据将预算和精力集中在转化率最高的平台。

## 技术架构说明（Claude Code 开发阶段参考）

### 需独立开发的模块

| 模块 | 技术方案 | 风险等级 |
|------|---------|---------|
| RPA社交平台发布 | Playwright/Selenium 模拟操作 | ⚠️ 高（平台反自动化检测，封号风险） |
| 多平台私信监控 | 各平台RPA定时抓取 | ⚠️ 高（同上） |
| 个人微信自动化 | WeChatFerry/itchat等逆向方案 | 🔴 极高（微信封号风险极高） |
| 企业微信集成 | 企业微信官方API | ✅ 低（官方支持，稳定合规） |

### 建议的开发顺序

1. **Phase 1**：最小可用系统落地（后端基础骨架 + 线索库 + 跟进记录 + AI服务后移 + 首页真实看板）
2. **Phase 2**：业务化与稳定化（缴费、学员、排课的真实数据化 + 日志、限流、测试、权限）
3. **Phase 3**：自动化增强（内容审核发布流、提醒任务、异步作业、外部系统集成）
4. **Phase 4**：高风险自动化验证（RPA自动发布、多平台私信监控、个人微信辅助）

### Phase 1 范围补充（2026-04-11 确认）

#### Phase 1 目标
将当前“前端原型 + mock 数据 + 前端直连 AI”的演示系统，升级为可供招生团队试运行的最小可用系统。

#### Phase 1 核心闭环
1. 新线索进入系统并统一入库
2. 招生专员查看线索详情并获取 AI 意向分析
3. AI 生成推荐跟进话术，供人工复制使用
4. 招生专员记录跟进结果，并同步更新线索状态、下一步动作和下次跟进时间
5. 对高意向线索生成并保存报价单/方案单，支持复制给顾问并回显
6. 对意向明确线索执行最小报名推进：确认报读院校/专业、报名进度、负责人备注
7. 对已报名线索登记首付款/全款信息，并记录下次催缴时间
8. 首页看板显示真实的线索、报名与待催缴数据

#### Phase 1 后端要求
- 新增后端服务，负责：
  - 线索管理 API
  - 跟进记录 API
  - AI 代理调用 API
  - 首页看板聚合 API
- Gemini API Key 仅保存在后端，前端不再直接读取或注入密钥
- Phase 1 可采用轻量技术栈（Node.js + TypeScript + Express + SQLite）

#### Phase 1 数据模型（最小版本）
至少包含以下数据实体：
- 用户（管理员/主管/招生专员）
- 线索（来源平台、昵称、联系方式、意向等级、当前状态、负责人）
- 跟进记录（沟通方式、内容、下一步动作、下次跟进时间）
- 报价单/方案单记录（线索ID、推荐院校、推荐专业、学制、学费、缴费方式、分期建议、适合人群、风险提示、复制文案、最后更新时间）
- 报名推进记录（意向院校、意向专业、报名进度、备注）
- 缴费记录（应收金额、已收金额、缴费方式、首付款时间、下次催缴时间）
- AI 调用记录（分析场景、输入、输出、时间）

#### Phase 1 线索状态（最小版本）
- 新线索
- 已联系
- 跟进中
- 意向明确
- 已报名
- 已流失

#### Phase 1 前端改造要求
- 内容工厂、线索管理、微信助手、首页看板优先接入真实 API
- 原有 mock 数据仅用于开发兜底，不再作为正式数据来源
- 微信助手在 Phase 1 仅实现“AI 话术辅助 + 人工沟通记录”，不做真实微信自动发送
- 招生专员可在微信助手中复制 AI 推荐话术或手动输入沟通内容，并将该内容记录为线索跟进记录，形成沟通留痕
- 首页看板在 Phase 1 优先展示真实的线索运营指标，如今日新线索、已联系线索、意向明确线索、待跟进线索、负责人表现、线索来源分布、近 7 天线索/跟进趋势
- 收入、缴费、在读人数等经营指标在 Phase 2 对应模块完成真实数据化前，不作为 Phase 1 首页核心验收项

#### Phase 1 验收标准
满足以下条件即可视为完成：
- 可创建、查询、更新线索
- 可新增并查看跟进记录
- 可为高意向线索创建、编辑、查看报价单/方案单
- 方案单字段完整展示并可保存回显
- 支持一键复制完整方案文案给顾问
- 可为线索登记报名推进信息（院校、专业、报名阶段）
- 可登记首付款/全款与下次催缴时间
- AI 意向分析与话术推荐改为通过后端接口调用
- 首页核心指标改为真实线索运营数据统计
- 首页不再依赖 mock 的收入、在读学员等非 Phase 1 数据作为核心判断依据
- 前端构建产物中不再暴露 Gemini API Key
- 项目具备最小运行文档与基础测试能力

> ⚠️ **风险提示**：RPA和个人微信自动化存在被平台封号的风险，建议：
> - RPA账号与主营账号隔离，使用专门的发布号
> - 个人微信部分建议以"AI生成话术 + 人工发送"为主，减少自动化操作
> - 优先使用企业微信官方API，稳定合规
